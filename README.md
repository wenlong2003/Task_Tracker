# 🗂️ Task Tracker

Task Tracker is a full-stack web application designed to help users efficiently organize and manage their daily tasks. The application provides a simple and intuitive interface where users can create an account, securely log in, and keep track of their responsibilities in a personalized dashboard.

The main goal of this project is to provide a clean and practical solution for task management while also demonstrating the implementation of a full-stack architecture. The system integrates a modern frontend interface with a backend API and a database to store and manage user data.

This project focuses on combining usability, performance, and simplicity so users can easily keep track of their tasks without unnecessary complexity.

---

# 🚀 Features

## 🔐 User Authentication
The application includes a secure user authentication system that allows individuals to create accounts and access their personal task dashboard.

- Users can register new accounts
- Secure login system
- Authentication verification before accessing protected pages
- User-specific task management

This ensures that every user has their own private workspace to manage their tasks.

---

## 📝 Task Management
The core functionality of the application revolves around organizing tasks in a clear and simple way.

Users are able to:

- Add new tasks
- View their tasks in a structured layout
- Keep track of responsibilities in a centralized dashboard

The goal of this feature is to help users stay organized and maintain productivity throughout their day.

## 🎨 User Interface
The project includes a clean and user-friendly interface designed to make task management simple and intuitive.

Key design goals include:

# 🛠️ Technologies Used

## ⚛️ Frontend
The frontend of the application is responsible for the user interface and user interactions.

Technologies used:

- React – For building interactive UI components
- Vite – For fast development and build performance
- CSS – For styling and layout design

These technologies allow the application to provide a fast and responsive experience.

---

## 🖥️ Backend
The backend manages the core logic of the application and handles communication between the frontend and the database.

Responsibilities include:

- Processing authentication requests
- Handling API endpoints
- Managing task-related data operations
- Communicating with the database

The backend ensures that user data is processed securely and efficiently.

---

## 🗄️ Database
The application uses a relational database to store user information and application data.

- MySQL is used to store user credentials and related data
- Data is structured and organized to support authentication and task management features

The database plays a critical role in ensuring that user data is persistent and accessible whenever the user logs into the system.

---

# 📂 Project Structure

The project is organized into separate sections to maintain a clear separation between the frontend and backend components.

frontend/  
Contains the React application responsible for the user interface and user interaction.

backend/  
Contains the server logic and API endpoints responsible for handling requests and communicating with the database.

This separation allows for better maintainability and scalability of the application.

# ▶️ Getting Started

To run this project locally, follow these steps.

## Clone the repository

git clone https://github.com/your-username/task-tracker.git

cd task-tracker

## Install dependencies

Frontend:

npm install

Backend:

pip install -r requirements.txt

## Run the application

Start the backend server.

python app.py

Then start the frontend development server.

npm run dev

Once both services are running, the application can be accessed in the browser.
